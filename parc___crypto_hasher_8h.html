Detailed Description

Computes digests of bytes or PARCBuffers.

The PARCCryptoHasher computes digests of bytes or PARCBuffers. It produces a PARCCryptoHash (without the "er"), which contains the digest and the algorithm used to compute the digest.

PARCCryptoHasherInterface  functor

Creates a custom hasher using the provided functor.

Useful for implementing HMAC [RFC 2104] without leaking the key outside the keystore.

The functor may carry an environment (i.e. info from the keystore) that will be echod back when CryptoHasher call the functor->hasher_setup(functor->functor_env). All subsequent calls are passed the setup context

Example:

1 {
2  static PARCCryptoHasherInterface hash_crc32c_template = {
3  .functor_env = NULL,
4  .hasher_setup = crc32cHasher_setup,
5  .hasher_init = crc32cHasher_init,
6  .hasher_update = crc32cHasher_update,
7  .hasher_finalize= crc32cHasher_finalize,
8  .hasher_destroy = crc32cHasher_destroy
9  };

void parcCryptoHasher_Release ( PARCCryptoHasher **  hasherP)

Release a previously acquired reference to the specified instance, decrementing the reference count for the instance.

Destroy the digester, releasing internal references as needed.

The pointer to the instance is set to NULL as a side-effect of this function.

If the invocation causes the last reference to the instance to be released, the instance is deallocated and the instance's implementation will perform additional cleanup and release other privately held references.

Parameters
[in,out]hasherPA pointer to a pointer to the instance to release.
